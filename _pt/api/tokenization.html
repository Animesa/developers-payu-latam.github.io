O produto Tokenização permite armazenar os dados dos cartões de crédito de seus clientes de forma segura, por meio da criação de um token com o qual você poderá realizar cobranças periodicamente ou implementar a funcionalidade de pagamento en 1 clique. Seguindo assim os padrões de segurança na manipulação dos dados de cartões de crédito PCI DSS (Payment Card Industry Data Security Standard).
Para usar as funcionalidades do produto de Tokenização a partir do API, você deve ter este serviço habilitado na sua conta PayU, se desejar adquiri-lo entre em contato no sac@payulatam.com.  
 
Como funciona?  
O produto “Tokenização” inclui dois processos independentes:

 
 
Os métodos disponíveis para a tokenização são: Registro individual de cartão de crédito, Registro massivo de cartões de crédito, Cobrança individual com token, Cobranças massivas com token, Consulta de token e Exclusão de token. A seguir, você encontrará a descrição de cada um.

Registro massivo de cartões de crédito
 
Mediante esta funcionalidade você pode registrar os dados dos cartões de crédito de vários clientes e assim obter os códigos sequenciais token.
Para tokenizar cartões de crédito massivamente você deve criar um arquivo com formato CSV sob as seguintes condições:  
  1. Cada registro de cartão de crédito a tokenizar, deve conter os dados para o registro de um cartão de crédito separados por vírgula na ordem a seguir: ID Pagador, nomes e sobrenomes, número de cartão de crédito, data de expiração, franquia, e número de identificação.
  2. O arquivo não deve ter cabeçalho. Na primeira linha constará o primeiro registro.
  3. O arquivo deve estar codificado sob o padrão UTF-8.
  4. O arquivo não deve ter mais de 10.000 registros..
 

  • JSON
  • XML
Exemplo pedido: 
{
   "language":"es",
   "command":"CREATE_BATCH_TOKENS",
   "merchant":{
      "apiLogin":"11959c415b33d0c",
      "apiKey":"6u39nqhq8ftd0hlvnjfs66eh8c"
   },
      "contentFile":"MDAxLG5vbWJyZSBhcGVsbGlkbyAxLDQxMTExMTExMTExMTExMTEsMjAxNS8wMSxWSVNBLDEyMzQ1Ng=="
}
 
Exemplo resposta: 
{
   "code": "SUCCESS",
   "error": null,
   "id": "edd187bb-067c-4016-b94d-3a4727cc86eb"
}
 
Você deve utilizar uma funcionalidade que pegue o arquivo e o mude a base 64, o resultado será o dado que se envia no parâmetro <contentFile>.

</merchant>
<contentFile>
MDAxLG5vbWJyZSBhcGVsbGlkbyAxLDQxMTExMTExMTExMTExMTEsMjAxNS8wMSxWSVNBLDEyMzQ1Ng==
</contentFile>
</request>

Cobranças massivas com token
 
Esta funcionalidade permitirá realizar cobranças massivas, utilizando os Tokens que foram previamente criados por nosso sistema.
Para esta funcionalidade você deve criar um arquivo com formato CSV, sob as seguintes condições:  
  1. Cada registro de transação a processar, deve conter os dados para seu processamento separados por vírgula e na seguinte ordem: ID Conta (identificador da conta virtual no PayU Latam), Token, código de segurança, número de parcelas, descrição da venda, e-mail do comprador, moeda (codificação ISO), valor total (imposto incluído), imposto, valor base devolução, valor adicional, idioma (codificação ISO).
  2. O arquivo não deve ter cabeçalho. Na primeira linha constará o primeiro registro.
  3. O arquivo deve estar codificado sob o padrão UTF-8.
  4. O arquivo não deve ter mais de 10.000 registros.

  • XML

  • XML

{
   "language":"es",

   "merchant":{
      "apiLogin":"11959c415b33d0c",
      "apiKey":"6u39nqhq8ftd0hlvnjfs66eh8c"
   },
   "removeCreditCardToken":{
      "payerId":"10",
      "creditCardTokenId":"32fa0a2d-ea79-40f3-8027-f9363296dd0e"

}
