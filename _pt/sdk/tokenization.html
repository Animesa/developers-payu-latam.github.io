O produto Tokenização permite armazenar los dados dos cartões de crédito de seus clientes de forma segura, por meio da criação de um token com o qual você poderá realizar cobranças periodicamente ou implementar a funcionalidade de pagamento em 1 clique. Seguindo assim os padrões de segurança na manipulação dos dados de cartões de crédito PCI DSS (Payment Card Industry Data Security Standard).
Para o uso das funcionalidades do produto de Tokenização a partir do API, você deve ter este serviço habilitado na sua conta PayU, se desejar adquiri-lo entre em contato com sac@payulatam.com.  
 
Como funciona?  
O produto “Tokenização” inclui dois processos independentes:

 
 
Os métodos disponíveis para a tokenização são: REGISTRO INDIVIDUAL DE CARTÃO DE CRÉDITO, COBRANÇA INDIVIDUAL COM TOKEN, CONSULTA DE TOKEN e EXCLUSÃO DE TOKEN. A seguir, você encontrará a descrição de cada um.  
 
 
REGISTRO INDIVIDUAL DE CARTÃO DE CRÉDITO
 
Mediante esta funcionalidade você pode registrar os dados do cartão de crédito de um cliente e assim obter um código sequencial token.
  • SDK Java
  • SDK PHP
Exemplo pedido: 
// -- Operação "Criar Token" --
Map<String, String> parameters = new HashMap<String, String>();
//Insira aqui o nome do pagador.
parameters.put(PayU.PARAMETERS.PAYER_NAME, "full name");
//Insira aqui o identificador do pagador.
parameters.put(PayU.PARAMETERS.PAYER_ID, "10");
//Insira aqui o documento de identificação do comprador.
parameters.put(PayU.PARAMETERS.PAYER_DNI, "32144457");
//Insira aqui o número do cartão de crédito
parameters.put(PayU.PARAMETERS.CREDIT_CARD_NUMBER, "4111111111111111");
//Insira aqui a data de vencimento do cartão de crédito
parameters.put(PayU.PARAMETERS.CREDIT_CARD_EXPIRATION_DATE, "2014/06");
//Insira aqui o nome do cartão de crédito
parameters.put(PayU.PARAMETERS.PAYMENT_METHOD,PaymentMethod.VISA.name() );
CreditCardToken response = PayUTokens.create(parameters);

if(response != null){
	response.getTokenId();
	response.getMaskedNumber();
	response.getPayerId();
	response.getIdentificationNumber();
	response.getPaymentMethod();
}

COBRANÇA INDIVIDUAL COM TOKEN
 
Esta funcionalidade permite realizar cobranças, utilizando um código Token que foi previamente criado por nosso sistema, e com o qual foram armazenados os dados dos cartões de crédito de seus clientes de forma segura.
  • Brasil
  • Colômbia
  • México
Cartões de crédito (“Autorização” ou “Autorização e Captura”)
Franquias disponíveis: Visa, Mastercard, Amex, Diners, ELO e Hipercard.
  • SDK Java
  • SDK PHP
Exemplo pedido:

CONSULTA DE TOKEN
 
Com esta funcionalidade você pode realizar a consulta dos Token de cartões de crédito previamente.
  • SDK Java
  • SDK PHP
Exemplo pedido: 
// -- Operação "Consultar token" --
Map<String, String> parameters = new HashMap<String, String>();
// -- Parâmetros opcionais --
//Insira aqui o identificador do pagador.
parameters.put(PayU.PARAMETERS.PAYER_ID, "10");
//Insira aqui o identificador do token.
parameters.put(PayU.PARAMETERS.TOKEN_ID, "3ba2c031-a8c0-4c9f-9025-7eacf8dd14e2");
//Insira aqui a data inicial a partir de onde filtrar com a data final até onde filtrar.
parameters.put(PayU.PARAMETERS.START_DATE, "2010-01-01T12:00:00");
parameters.put(PayU.PARAMETERS.END_DATE, "2012-01-01T12:00:00");

List<CreditCardToken> response = PayUTokens.find(parameters);
Iterator<CreditCardToken> tokens_iterator=response.iterator();

while(tokens_iterator.hasNext()){
	CreditCardToken token= (CreditCardToken) tokens_iterator.next();

	token.getTokenId();
	token.getMaskedNumber();
	token.getPayerId();
	token.getIdentificationNumber();
	token.getPaymentMethod();

}
