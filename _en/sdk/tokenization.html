The Tokenization product allows you to store your customers’ credit card data safely, through the creation of a token with which you can make regular charges or implement the 1 Click payment functionality, thus following the PCI DSS (Payment Card Industry Data Security Standard) credit card data security and safe handling standards.
To use the functionality of the product from the Tokenization API, you must have this service enabled on your PayU account, if you want to purchase contact sac@payulatam.com.  
 
How does it work?  
The product "Tokenization" includes two separate processes:

 
 
Available methods for tokenization are: Individual credit card registration, Individual charge with token, Query of token and Elimination of token. Below is a description of each.  
 
 
Individual credit card registration
 
Using this feature you can record data from a customer’s credit card and get a token sequence number.
  • SDK Java
  • SDK PHP
Example request: 
//-- “Create Token” operation--
Map<String, String> parameters = new HashMap<String, String>();
//Enter the payer's name here.
parameters.put(PayU.PARAMETERS.PAYER_NAME, "full name");
//Enter the payer's ID here.
parameters.put(PayU.PARAMETERS.PAYER_ID, "10");
//Enter the payer's contact document here.
parameters.put(PayU.PARAMETERS.PAYER_DNI, "32144457");
//Enter the number of the credit card here
parameters.put(PayU.PARAMETERS.CREDIT_CARD_NUMBER, "4111111111111111");
//Enter the expiration date of the credit card here
parameters.put(PayU.PARAMETERS.CREDIT_CARD_EXPIRATION_DATE, "2014/06");
//Enter the name of the credit card here
parameters.put(PayU.PARAMETERS.PAYMENT_METHOD, "VISA");
CreditCardToken response = PayUTokens.create(parameters);

if(response != null){
	response.getTokenId();
	response.getMaskedNumber();
	response.getPayerId();
	response.getIdentificationNumber();
	response.getPaymentMethod();
}

Individual charge with token
 
This feature allows you to charge using a Token code previously created by our system, and with which your customers’ credit card data are safely stored.
  • Brazil
  • Colombia
  • Mexico
  • Peru
Credit cards charges with token (“Authorization” or “Authorization and Capture”)
Available franchises: Visa, Mastercard, Amex, Diners, ELO and Hipercard.
