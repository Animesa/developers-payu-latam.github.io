Webcheckout Integration

  • What should you know?
  • Integration components
  • How does it work?
 
 
 
 
This type of integration allows your customer to select the items he wishes to purchase and at the time of purchase to be redirected to our payment gateway (webcheckout).

You can include your company logo on this page. Data exchange is via the HTTP POST method.

It is necessary that the developer has expertise integrating a dynamic programming language type like PHP, Java, etc.

 
 
INTEGRATION COMPONENTS  
 
1. Payment form  With this HTML form you send the transaction request our gateway, including the purchase payment data, using the HTTP POST method.  
2. Response page  It is a page that shows the result of the transaction to the payer. Our system will direct the payer to this page once the transaction is performed. The data with the payment results will be sent to your system via HTTP GET. This page is called with all states of transactions: approved, rejected, in validation, awaiting payment (for cash), etc. This page is mandatory.  
3. Confirmation page  Through this page we confirm your system the end result of the transaction. At this point you can update your systemâ€™s inventories, orders, or databases. This page will never be visible to the buyer, since its goal is to enable communication between systems. The data are sent via the HTTP POST method. This page is only called for approved and rejected states. This page is not mandatory.  

 
 
How does the webcheckout integration work?  
 
01. The process starts when the buyer comes to your WEB page, selects the products or services he wishes to purchase and clicks the pay button. Your system must add up the value of the purchase and send a payment form to our system through the HTTP POST method with some purchase data.

02. The payer is directed to our checkout, there he can choose the payment means with which he wants to perform the transaction.

03. PayU processes the transaction and displays the result in the response page.

04. PayU redirects the payer to your web page (response page) and sends the information of the result to your system via the HTTP GET method.

05. The system will notify the final status of the transaction through a confirmation page via the HTTP POST method, additionally the system will send you and each payer a confirmation email.

 
 
Consider:
  1. The coding of your page should be UTF-8.
  2. You must not enter our Web Checkout within an iframe
  3. You must not mask the URL during the checkout process
  4. If you have a SSL certificate on your confirmation page, it is important that you send it to sac@payulatam.com requesting that it be loaded into our bank of certificates.
