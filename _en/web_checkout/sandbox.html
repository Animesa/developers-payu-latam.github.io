Example: 
merchantId: 500238
ApiKey: 6u39nqhq8ftd0hlvnjfs66eh8c
referenceCode: TestPayU
accountId: 500537
description: Test PAYU
amount: 3
tax: 0
taxReturnBase: 0
currency: USD
signature: be2f083cb3391c84fdf5fd6176801278
test: 1
buyerEmail: test@test.com
 
Pointing to the following URL:
https://stg.gateway.payulatam.com/ppp-web-gateway
 
 
Note:
  1. It is important that during the performance of the test you do not use real credit card data, you can use credit card generators.
  2. Include the field test = 1 to indicate the system that it is a test transaction.
  3. Testing can be performed only with credit card payment means, cash or bank transfers cannot be tested with this method.
  4. You cannot perform tests with Peru’s payment means, since these franchises have 3D security.
 
 
You'll get access to the payment gateway corresponding to the country of the account you selected for testing, in order to perform the test you must select one of the available credit card franchises.

 
Once you are in the credit card payment form, and ensuring that you have the message at the top of the Transaction gateway in test mode, you must:
  1. Enter the text APPROVED in the Full Name field if you want the transaction to be approved, or REJECTED if you want it to be rejected.
  2. You must enter a valid card number in the Card Number field and which corresponds to the selected franchise. For this you can find an online credit card generator.
  3. All other fields can be random.

 
A response is obtained, in this case of successful completion of the movement.

 
 
 
CHANGE TO PRODUCTION  
After you perform all the tests in the system and verify that everything is working correctly, you can change to the production server; where the actual transactions will be processed.  
To do this you should uncheck the Transaction in test mode option that you will find in your administrative module.  

 
Additionally you must create the payment forms with the following parameters corresponding to your account:  
merchantId: of your account
ApiKey: of your account
referenceCode: unique sale’s reference for each transaction
accountId: of your account
description: description of the sale
amount: value of the sale 
tax: tax value associated with the sale
taxReturnBase: value of tax refunds
currency: the currency of your account
signature: the signature which is generated for each transaction
test: 0
buyerEmail: buyer’s email
 
Pointing to the production URL:
https://gateway.payulatam.com/ppp-web-gateway
