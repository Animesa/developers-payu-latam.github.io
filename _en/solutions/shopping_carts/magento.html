MAGENTO

Requirements:
  • Have an active account in PayU Latam.
  • Available for Magento versions from 1.7.x to 1.9.x.
  • Access to magento instalation folders.
  • Access to magento Backend
 
Installation:  
 
01. Clean Magento Cache: go to System > Cache Management and then click "Flush Magento Cache".
02. Download the PayU Plugin for Magento.
Unzip the file Plugin_PayU_Magento-1.4.zip. the uncompressed folder is called "app" y contains:

       a. code
       b. design
       c. etc.
Download plugin
03. Copy the "Gfgrisales" folder located in "app/code/local" in the path "app/code/local" located in the root of Magento installation. If the "local" folder does not exist in the path of Magento, you must create it and then copy the listed files. Make sure all the folders are all read/write enabled.
04. Copy the "payu" folder of the "app/design/frontend/base/default/template/payu" path of the decompressed file, inside of the "app/design/frontend/base/default/template" folder located inthe root of the Magento installation.
05. Copy the file "Gfgrisales_Payu.xml" located in "app/etc/modules" of the uncompressed file. This must be copied into the "app/etc/modules" folder, located in the root of the Magento installation.
 
 
Set up:  
 
01. Check the plugin is enabled: Enter the Magento administrator and go to the System> Configuration> Advanced> Advanced menu. Click on "Disable Modules Output" and check the PayU extensión "Gfgrisales_Payu" is enabled.
02. Then, while being in Setup, click "Sales> Payment Methods" where PayU’s payment method should appear. To display the tab click on it.

Testing statuses
Once you are in the credit card payment form, and ensuring that you have the message at the top of the gateway "Transaction in test mode", you must:

  1. Enter the text APPROVED in the Full Name field if you want the transaction to be approved, REJECTED if you want it to be rejected or PENDING if you want it to be pending.
  2. You must enter a valid card number in the Card Number field and which corresponds to the selected franchise. For this you can find an online credit card generator.
  3. All other fields can be random.

Consider:
  • Approved payments will have a processing status in the orders report.
  • Once the payment is received correctly, Magento will create an invoice with the associated order.
