API INTEGRATION - REFUNDS MODULE

  • How does it work?
  • Refund request
  • Query response

This feature allows you to request the cancellation and refund of transactions that were paid with credit card; and which for reasons of dissatisfaction of the buyer, multiple charges in the same purchase, or when the buyer requires the refund because he did not receive the product or service.

You can make the refund request using the Refund or Void method which is provided for this purpose. Once the request is made, a customer service agent from PayU takes charge of validating the information and justification of the request to approve the request and to perform the refund or refuse the request if it does not meet the requirements.
Additionally, you can check the status of your refund request using the Query API available for that purpose.

Consider:
  • Captured state that have been made with a credit card. If the request refers to a different payment means (cash payment means, bank transfer), the application will be declined by PayU´s customer service agent.

  • The system will only create one application for each refund request, if the request is repeatedly posted for the same transaction, the system will only record one request in the module and will indicate that the request is already registered.

  • If an order or transaction has already been refunded, the system will indicate that it is not possible to make the request again. PayU only accepts reimbursement requests of transactions that are in the Capturedstate.

  • If a reimbursement request of an order or transaction was previously Declined, you can retry the reimbursement request if you consider it necessary.

  • Once you make the request, the amount of the transaction becomes part of the Frozen Balance of your PayU account until response to the request is given. If your refund request is Approved, since it is refunded to the payer. If the refund request is Ddeclined, the value of the transaction will be released from the Frozen Balance and will return to the available Balance of your PayU account.
 
 
REQUEST REFUND  

If a transaction is in the “Captured”, state, you can use the Refund method to perform the refund request.

It is compulsorily that you submit the following parameters:
  • 1. Order’s ID.
  • 2. Transaction’s ID.
  • 3. The type of request which will be REFUND.
  • 4. Reason for requesting the refund or cancellation of the transaction.
Request example:
  • JSON
  • XML
 
{
	"language": "es",
	"command": "SUBMIT_TRANSACTION",
	"merchant": {
		"apiKey": "P4yU*********L4t4mC0l",
		"apiLogin": "CoM3r*******c1o5"
	},
	"transaction": {
		"order": {
			"id": "45678901"
		},
		"type": "REFUND",
		"reason": "Reason for requesting the refund or cancellation of the transaction",
		"parentTransactionId": "8795a432-12ab-34cd-1234-a9876bc54321"
	},
	"test": false
}

CHECK RESPONSE  

 
Browse through PayU’s Administrative Module.

In order to perform the query using the administrative module, you must enter your username and password to access the Reportes tab, access Orders, in the lateral menu, and enter the ID of the order to be consulted in the search filters.
The system will submit the order information, and display the associated transactions. Once your refund request is approved or declined, in the order a new record type Refund will be shown with its respective state, either Approved or Declined, according to the solution given by the customer Service agent.

  • Unresolved request
  • Approved request
  • Request declined
