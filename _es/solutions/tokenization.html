Módulo administrativo - tokenización

  • Registro masivo
  • Cobro individual
  • Cobro masivo
  • Consulta

El producto Tokenizacion te permite almacenar los datos de las tarjetas de crédito de tus clientes de forma segura, por medio de la creación de un token con el cuál podrás realizar cobros periódicamente o implementar la funcionalidad de pago en 1 Clic. Esta funcionalidad no se encuentra en la configuración inicial de tu Módulo administrativo, por lo tanto debes contactar tu asesor comercial o escribir a sac@payulatam.com y pedir la activación de la misma. Una vez sea activada la funcionalidad tendrás acceso a la pestaña en el módulo administrativo “Cobros con token”.
 
¿Cómo funciona?  
El producto “Tokenizacion” incluye dos procesos independientes:

 
 
 
Registro individual de tarjeta de crédito  

Esta funcionalidad te permitirá ingresar individualmente los datos de la tarjeta de crédito de un cliente pagador, el sistema encriptará estos datos y generará un código token con el cuál podrás realizar cobros de forma segura.
Para realizar el proceso de tokenización debes contar con estos datos:

  • ID pagador (identificador único del pagador en tu comercio).
  • Nombres y Apellidos (Datos del tarjetahabiente, deben diligenciarse tal y como aparecen en la tarjeta de crédito).
  • Franquicia. Ej.: Visa, MasterCard, American Express, etc.
  • Número de la tarjeta de crédito.
  • Fecha de expiración de la tarjeta de crédito.
  • Número de identificación del cliente pagador.
 

01. Para registrar una tarjeta de crédito y obtener el "Token", debes ingresar a “Cobros con token” y luego a “Registro tarjeta crédito”, diligenciar los campos solicitados y dar clic al botón enviar.

 

02. El sistema toma la información diligenciada y realiza el proceso de registro de la tarjeta de crédito, generando y devolviendo como respuesta el “token”, el número visible de la tarjeta de crédito, la franquicia y el id del pagador.

 
 
Registro masivo de tarjetas de crédito  
Esta funcionalidad te permitirá ingresar un lote de hasta 10.000 registros que incluya los datos de las tarjetas de crédito de tus clientes, el sistema encriptará estos datos y generará los códigos token correspondientes, con los que podrás realizar cobros de forma segura.

Para usar esta funcionalidad debes crear un archivo con formato CSV bajo las siguientes condiciones:

  • Cada registro de tarjeta de crédito a tokenizar, debe contener los datos para el registro de una tarjeta de crédito separados por coma y en el siguiente orden:
• ID pagador (identificador único del pagador en tu comercio)
• Nombres y Apellidos (Datos del tarjetahabiente, deben diligenciarse tal y como aparecen en la tarjeta de crédito).
• Número de tarjeta de crédito.
• Fecha de expiración.
• Franquicia. Ej.: Visa, MasterCard, American Express, etc.
• Número de identificación.
  • El archivo no debe tener encabezado. En la primera línea va el primer registro.
  • El archivo debe estar codificado bajo el estándar UTF-8.
  • El archivo no debe tener más de 10000 registros.

01. Para realizar el registro masivo de tarjetas de crédito y obtener el "Token", para cada una de ellas, debes ingresar a “Cobros con token” y luego a “Reg masivo tarjeta crédito”, luego dar clic al botón adjuntar y seleccionar el archivo CSV con los registros de tarjetas de crédito a tokenizar.

Seleccionando el archivo para adjuntar

02. Al adjuntar un archivo el sistema lo toma y genera un nuevo registro de lote en procesamiento con la siguiente información: • Número de lote: identificador único del lote generado por el sistema a partir de un archivo de tarjetas de crédito a tokenizar
• Fecha de creación: fecha en que se sube el archivo.
• Fecha de procesamiento: fecha en que el sistema procesa completamente el archivo.

03.Una vez que el lote sea correctamente procesado, puedes consultar los token resultantes haciendo clic en la columna detalle link “Ver” y el sistema te devolverá un archivo en formato CSV que contiene: • Token generado para la tarjeta de crédito
• ID pagador: identificador único del pagador en tu comercio.
• Franquicia Ej. visa, MasterCard, American Express, etc.
• Número visible de la tarjeta de crédito
• Estado
• Nombre tarjeta habiente
• Documento tarjeta habiente
• Fecha de creación
• ID Comercio
• Número de lote
• Descripción de error

 
 
Cobro individual con token  

Esta funcionalidad te permitirá realizar cobros, utilizando un código Token que fue previamente creado por nuestro sistema, y con el cual se almacenaron los datos de las tarjetas de crédito de tus clientes de forma segura.

Para esta funcionalidad debes contar con los siguientes datos:

  • ID cuenta: identificador de la cuenta virtual en PayU Latam a la que se debe asociar la transacción.
  • Código secuencial de la tarjeta de crédito “Token”
  • Código de seguridad: este campo es opcional y solo se requerirá en caso de que las entidades financieras nos obliguen a enviarlo.
  • Número de cuotas: se puede definir desde 1 hasta 12 cuotas.
  • E-mail del comprador: este dato es opcional, quedará asociado a la información de la transacción y se utilizará para envío de notificación del cobro realizado.
  • Referencia de venta: aplican las restricciones de PayU Latam para comercios que manejen esquema de validación de referencia única.
  • Descripción de la venta.
  • Moneda: El sistema carga la codificación ISO de la moneda local según el país de la cuenta seleccionada.
  • Valor Total: debe incluir el impuesto.
  • Impuesto.
  • Base devolución del impuesto.
  • Valor adicional.
  • Idioma: se toma en cuenta para el envío de las notificaciones en el proceso de compra.
 

01. Para realizar un cobro en línea, utilizando el Token de una tarjeta de crédito, debes ingresar a “Cobros con token” y luego a “Cobro en línea”, diligenciar los campos solicitados y dar clic al botón enviar.

02. Luego de procesar la transacción el sistema muestra en pantalla del resultado:

 
Nota: Se debe tener en cuenta que al procesar una transacción el sistema enviará los correos de notificación a tu correo registrado y al correo del pagador, y reportará las páginas de respuesta y confirmación de acuerdo a la configuración de tu cuenta.
 
 
Cobros masivos con token  

Esta funcionalidad te permitirá realizar cobros, utilizando los códigos “Token” que fueron previamente creados por nuestro sistema, y con los cuales se almacenaron los datos de las tarjetas de crédito de tus clientes de forma segura.
Para esta funcionalidad debes crear un archivo con formato CSV bajo las siguientes condiciones:

  • Cada registro de transacción a procesar, debe contener los datos para su procesamiento separados por coma y en el siguiente orden,
• ID Cuenta: identificador de la cuenta virtual en PayU Latam a la que se debe asociar la transacción.
• Código secuencial de la tarjeta de crédito “Token”.
• Código de seguridad: Este campo es opcional en caso de que la red financiera lo solicite.
• Número de cuotas: se puede definir desde 1 hasta 12 cuotas.
• Referencia de venta.
• Descripción de la venta: aplican las restricciones de PayU Latam para comercios que manejen esquema de validación de referencia única.
• E-mail del comprador: este dato es opcional, quedará asociado a la información de la transacción y se utilizará para envío de notificación del cobro realizado.
• Moneda: codificación ISO de la moneda a utilizar en la transacción.
• Valor total: debe incluir el impuesto.
• Impuesto.
• Valor base devolución.
• Valor adicional.
• Idioma (ISO): se toma en cuenta para el envío de las notificaciones en el proceso de compra.
  • El archivo no debe tener encabezado. En la primera línea va el primer registro.
  • El archivo debe estar codificado bajo el estándar UTF-8.
  • El archivo no debe tener más de 10000 registros.
Ejemplo:

01. Para realizar cobros masivos en línea, debes ingresar a “Cobros con token” y luego a “Cobros masivos en línea”, dar clic al botón adjuntar y seleccionar el archivo CSV con los registros de transacciones a procesar.

Seleccionando el archivo para adjuntar:

02.Al adjuntar un archivo el sistema lo toma y genera un nuevo registro de lote en procesamiento con la siguiente información: • Número de lote: identificador único del lote generado por el sistema a partir de un archivo de tarjetas de crédito a tokenizar.
• Fecha de creación: fecha en que se sube el archivo.
• Fecha de procesamiento: fecha en que el sistema procesa completamente el archivo.
• Estado: Los lotes pueden encontrarse en los estados ERROR, PENDING, PROCESSING, PROCESSED.

Estados
ERROR El archivo adjunto contiene por lo menos un error en uno de sus registros, razón por la cual no es procesado. En este caso puedes dar clic en la columna detalle link “Errores” y obtendrás como respuesta un archivo que contiene el detalle de los errores encontrados en el archivo.
PENDING El archivo esta en espera para ser procesado por el sistema.
PROCESSING

 El archivo está siendo procesado por el sistema.
PROCESSED El archivo luego de ser validado no contienen ningún error y todos los registros de este son procesados por el sistema

03. Una vez el lote ha sido procesado, el sistema te entregará un archivo en formato CSV que contiene:
• ID pagador
• E-mail comprador
• Referencia
• Descripción de la venta
• Moneda
• Valor
• ID transacción: identificador de la transacción en PayU Latam
• Código autorización
• Fecha de procesamiento
• Estado
• Código de respuesta
• Número visible de la tarjeta de crédito
• Franquicia

 
 
Consulta de token  

Esta funcionalidad te permite realizar la consulta de los Token de tarjetas de crédito que has registrado.
La consulta se puede realizar a partir de los siguientes parámetros:

 
  • Id pagador: identificador único del pagador en tu comercio.
  • Código secuencial de la tarjeta de crédito “Token”
  • Rango de fechas (fecha inicial – fecha final)

01. Para realizar la consulta de uno o varios “Token” debes ingresar a “Cobros con token” y luego a “Gestión de token”, definir los parámetros de búsqueda deseados y dar clic en el botón consultar.

02. Al realizarse la consulta el sistema te devolverá un listado de todos los registros encontrados con la siguiente información: • Fecha de registro.
• ID pagador.
• Código secuencial de tarjeta de crédito Token.
• Número visible de la tarjeta de crédito.
• Franquicia.

 
 
Eliminación de token  
Esta funcionalidad te permite eliminar un registro de tarjeta de crédito tokenizada, de tal manera que no será posible volver a ejecutar cobros utilizando el código secuencial Token generado para ésta.

Para eliminar un token los datos solicitados son:

  • Id pagador: identificador único del pagador en tu comercio.
  • Código secuencial de la tarjeta de crédito “Token”.
 

01. Debes ingresar a “Cobros con token” y luego a “Gestión de token”, debes realizar la consulta del token que deseas eliminar y al ubicarlo en los resultados de la consulta dar clic en la columna opciones en el link “Dar de baja”.

02. Al dar clic sobre el link “Dar de baja”, el sistema te solicitará que confirmes la eliminación del token.

03. Al aceptar la acción, el sistema le confirma al usuario que la tarjeta de crédito ha sido eliminada (dada de baja).
