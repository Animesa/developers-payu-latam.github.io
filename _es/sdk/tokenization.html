El producto Tokenización te permite almacenar los datos de las tarjetas de crédito de tus clientes de forma segura, por medio de la creación de un token con el cuál podrás realizar cobros periódicamente o implementar la funcionalidad de pago en 1 Clic. Siguiendo así los estándares de seguridad en la manipulación de los datos de tarjetas de crédito PCI DSS (Payment Card Industry Data Security Standard).
Para el uso de las funcionalidades del producto de Tokenización desde el API, debes tener habilitado este servicio en tu cuenta PayU, si deseas adquirirlo comunícate con sac@payulatam.com.  
 
¿Cómo funciona?  
El producto “Tokenizacion” incluye dos procesos independientes:

 
 
Los métodos disponibles para la tokenización son: Registro individual de tarjeta de crédito, Cobro individual con token, consulta de token y Eliminación de token. A continuación encontrarás la descripción de cada uno.  
 
 
Registro individual de tarjeta de crédito
 
Mediante esta funcionalidad puedes registrar los datos de la tarjeta de crédito de un cliente y así obtener un código secuencial token.
  • SDK Java
  • SDK PHP
Ejemplo petición: 
// -- Operación "Crear Token" --
Map<String, String> parameters = new HashMap<String, String>();
//Ingrese aquí el nombre del pagador.
parameters.put(PayU.PARAMETERS.PAYER_NAME, "full name");
//Ingrese aquí el identificador del pagador.
parameters.put(PayU.PARAMETERS.PAYER_ID, "10");
//Ingrese aquí el documento de identificación del comprador.
parameters.put(PayU.PARAMETERS.PAYER_DNI, "32144457");
//Ingrese aquí el número de la tarjeta de crédito
parameters.put(PayU.PARAMETERS.CREDIT_CARD_NUMBER, "4111111111111111");
//Ingrese aquí la fecha de vencimiento de la tarjeta de crédito
parameters.put(PayU.PARAMETERS.CREDIT_CARD_EXPIRATION_DATE, "2014/06");
//Ingrese aquí el nombre de la tarjeta de crédito
parameters.put(PayU.PARAMETERS.PAYMENT_METHOD, "VISA");
CreditCardToken response = PayUTokens.create(parameters);

if(response != null){
	response.getTokenId();
	response.getMaskedNumber();
	response.getPayerId();
	response.getIdentificationNumber();
	response.getPaymentMethod();
}

Cobro individual con token
 
Esta funcionalidad te permite realizar cobros, utilizando un código Token que fue previamente creado por nuestro sistema, y con el cual se almacenaron los datos de las tarjetas de crédito de tus clientes de forma segura.
  • Brasil
  • Colombia
  • México
  • Perú
Tarjetas de crédito (“Autorización” o “Autorización y Captura”)
Franquicias disponibles: Visa, Mastercard, Amex, Diners y ELO
  • SDK Java
  • SDK PHP

Con esta funcionalidad puedes realizar la consulta de los Token de tarjetas de crédito previamente.
  • SDK Java
  • SDK PHP

// -- Operación "Consultar token" --

// -- Parámetros opcionales
//Ingresa aquí el identificador del pagador.
parameters.put(PayU.PARAMETERS.PAYER_ID, "10");

//Ingresa aquí la fecha inicial desde donde filtrar con la fecha final hasta donde filtrar.
parameters.put(PayU.PARAMETERS.START_DATE, "2010-01-01T12:00:00");
parameters.put(PayU.PARAMETERS.END_DATE, "2012-01-01T12:00:00");

List<CreditCardToken> response = PayUTokens.find(parameters);
Iterator<CreditCardToken> tokens_iterator=response.iterator();

while(tokens_iterator.hasNext()){
	CreditCardToken token= (CreditCardToken) tokens_iterator.next();

	token.getTokenId();
	token.getMaskedNumber();
	token.getPayerId();
	token.getIdentificationNumber();
	token.getPaymentMethod();

}
