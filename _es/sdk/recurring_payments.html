Los pagos recurrentes son cobros automatizados que se realizan periódicamente (diario, mensual, anual), de aquellos cargos por consumo de productos o servicios como membresías, suscripciones, pólizas o recibos con valor fijo; que previamente fueron autorizados por el cliente.  

 
Apuntando a las correspondientes URLs:
  • SDK Java
  • SDK PHP
PayU.paymentsUrl = “https://api.payulatam.com/payments-api/”;
PayU.reportsUrl = “https://api.payulatam.com/reports-api/”;
Environment::setPaymentsCustomUrl(“https://api.payulatam.com/payments-api/4.0/service.cgi”);/
Environment::setReportsCustomUrl(“https://api.payulatam.com/reports-api/4.0/service.cgi”);
Environment::setSubscriptionsCustomUrl(“https://api.payulatam.com/payments-api/rest/v4.3/”);
 
 
Los métodos disponibles para los pagos recurrentes son: Plan, Cliente, Tarjeta de crédito, Suscripción y Cargos adicionales. A continuación encontrarás la descripción de cada uno.
 
Plan
 
Mediante esta funcionalidad puedes registrar los datos de la tarjeta de crédito de un cliente y así obtener un código secuencial token.  
Se encuentran disponibles las siguientes operaciones:  
Url Metodo Descripción
/rest/v4.3/plans POST Creación de un nuevo plan para suscripciones asociado al comercio.
/rest/v4.3/plans/{planCode} PUT Actualizar la información de un plan para suscripciones.
{planCode} : Código de identificación del plan para el comercio.
/rest/v4.3/plans/{planCode} GET Consultar toda la información de un plan para suscripciones asociado al comercio.
{planCode} : Código de identificación del plan para el comercio.
/rest/v4.3/plans/{planCode} DELETE Eliminar todo un plan para suscripciones asociado al comercio.
{planCode} : Código de identificación del plan para el comercio.

Cliente
 
Un cliente es la unidad que identifica quien será el beneficiario de un producto o servicio prestado y que se encuentra asociado a un plan de pagos recurrentes.  
Se encuentran disponibles las siguientes operaciones:  
Url Metodo Descripción
/rest/v4.3/customers/ POST Creación de un cliente en el sistema.
/rest/v4.3/customers/{customerId} PUT Actualiza la información de un cliente en el sistema.
{customerId} : Identificador del cliente que se desea actualizar.
/rest/v4.3/customers/{customerId} GET Consulta la información relacionada con el cliente
{customerId} : Identificador del cliente del cual se desea conocer la información asociada.
/rest/v4.3/customers/{customerId} DELETE Elimina un usuario del sistema.
{customerId} : Identificador del cliente que se desea eliminar.
Creación:
  • SDK Java
  • SDK PHP
Ejemplo petición: 
Map<String, String> parameters = new HashMap<String, String>();

Tarjeta de crédito
 
Es el medio de pago con el cual se relaciona un Plan y un Pagador, se encuentra compuesto por el número de tarjeta de crédito (el cual será tokenizado para almacenar los datos de forma segura), la fecha de vencimiento de la tarjeta y algunos datos adicionales de dirección.  
Se encuentran disponibles las siguientes operaciones:  
Url Metodo Descripción
/rest/v4.3/customers/{customerID}/creditCards POST Creación de una tarjeta de crédito (Token) y asociarla a un usuario.
{customerId} : Identificador del cliente al cual se le va a asociar el token
/rest/v4.3/creditCards/{creditCardId} PUT Actualizar la información de un token.
{creditCardId} : Identificador del token que se desea actualizar.
/rest/v4.3/creditCards/{creditCardId} GET Consultar la información de una tarjeta de crédito (Token) dato su identificador.
{creditCardId} : Token de la tarjeta de crédito que desea consultarse
/rest/v4.3/customers/{customerID}/creditCards/{creditCardId} DELETE Eliminar una tarjeta de crédito (Token) asociado a un usuario.
{customerId} : Identificador del cliente al cual se le va a eliminar el token
{creditCardId} : Identificador del token que se desea eliminar

Suscripción
 
Una suscripción es la relación entre un plan de pagos, un pagador y una tarjeta de crédito y es el elemento con el que se controla la ejecución de los cobros correspondientes.  
Se encuentran disponibles las siguientes operaciones:  
Url Metodo Descripción
/rest/v4.3/subscriptions/ POST Creación de una nueva suscripción de un cliente a un plan.
/rest/v4.3/subscriptions/{subscriptionId} PUT Actualizar la información asociada a la suscripción indicada. En este momento sólo es posible actualizar el token de la tarjeta de crédito a la cual se realiza el cargo de la suscripción.
{subscriptionId} : Identificación de la suscripción
/rest/v4.3/subscriptions/{subscriptionId} GET Consultar la información básica asociada a la suscripción indicada.
{subscriptionId} : Identificación de la suscripción
/rest/v4.3/subscriptions/{subscriptionId} DELETE Eliminar la suscripción, es decir la relación del cliente al plan.

Cargos adicionales
 
Un cargo puede ser un cobro adicional o un descuento realizado sobre el valor de uno de los pagos que conforman el plan de pagos recurrentes. Estos solo afectan el siguiente cobro pendiente y se ejecutan una única vez.  
Se encuentran disponibles las siguientes operaciones:  
Url Metodo DESCRIPCIÓN
/rest/v4.3/subscriptions/{subscriptionId}/recurringBillItems POST Adiciona cargos extras a la factura correspondiente al periodo actual.
{subscriptionId} : Identificación de la suscripción
/rest/v4.3/recurringBillItems/{recurringBillItemId} PUT Actualiza la información del cargo extra de una factura
{recurringBillItemId} : Identificador del cargo extra
/rest/v4.3/recurringBillItems/{recurringBillItemId} GET Consulta la información del cargo extra de una factura a partir de su identificador.
{recurringBillItemId} : Identificador del cargo extra
/rest/v4.3/recurringBillItems/{recurringBillItemId} DELETE Eliminar un cargo extra de una factura
{recurringBillItemId} : Identificador del cargo extra
/rest/v4.3/recurringBillItems/ GET Consulta de los cargos extras de las facturas del comercio que cumplen con los filtros estipulados. Los filtros disponibles se muestra a continuación y deben ser enviados como named parameters dentro de la URL:
{subscriptionId} : Identificación de la suscripción
{description} : Descripción ingresada en el cargo extra
