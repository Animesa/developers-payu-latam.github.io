Buenas prácticas para una buena integración

Ten en cuenta las siguientes recomendaciones para llevar a cabo el desarrollo de una integracion exitosa con PayU.

El envío de la mayor cantidad de variables a nuestra plataforma es la mejor práctica, por eso no olvides enviar las siguientes variables en tus peticiones.

  1. Merchant ID
  2. Account ID
  3. Api Key y Api Login
  4. Firma digital
  5. Referencia de venta
  6. Descripción
  7. Valor
  8. Moneda
  9. Nombre Completo
  10. Correo Electrónico
  11. Teléfono
  12. Numero de documento
  13. Dirección

Es importante que envíes en la petición XML/JSON los datos del usuario que está realizando la compra. Con esta información, nuestro módulo antifraude tendrá mayores criterios para realizar la correcta validación de las transacciones.

Es necesario que adicional envíes la información del dispositivo desde el cual se está realizando el pago, a continuación te relacionamos las variables mínimas obligatorias.

  1. Direccion IP: Identificador de corto plazo para las máquinas desde donde se realizan las transacciones.
  2. Cookie: Es utilizada para hacer seguimiento y relacionamiento de los comportamientos de los pagadores respecto a la máquina. Teniendo en cuenta la facilidad del cambio, resulta ser un buen indicador para determinar a compradores reales.
  3. Device ID: Es un código que captura la información del equipo desde donde se genera la transacción y crea un identificador único para cada equipo. Esta herramienta permite detectar atacantes.
  4. User Agent: Es el identificador del navegador desde donde se hace la transacción. Ejemplo: Mozilla/5.0 (Windows NT 6.1; WOW64; Trident/7.0; rv:11.0) like Gecko)

Brasil  
Medios de pago  
 
Variables requeridas
  1. accountId: Esta variable de tipo numérico, se incluye en el formulario de pago e identifica la cuenta del usuario para cada país que tenga asociado el comercio, al enviarla se despliegan solo los medios de pago pertenecientes a dicho país.

Colombia  
Medios de pago  
 
Variables requeridas
  1. CUS: Es una variable que se envía en la página de respuesta; código único de seguimiento, es la referencia del pago dentro del Banco, aplica solo para pagos con PSE.
  2. pseBank: Es una variable que se envía en la página de respuesta; es el nombre del banco, aplica solo para pagos con PSE.
  3. pseCycle: Es una variable que se envía en la página de respuesta, es el Identificador generado por PSE.
  4. pseReference1 / pseReference2 / pseReference3:: Son variables que se envían en la página de respuesta, son los números de referencias para pagos con PSE.
 
 
 
México  
Medios de pago  
 
 
Para México debes tener en cuenta
  1. Para usar meses sin intereses debes hacer uso de nuestra API de promociones
  2. Debes enviar la variable BirthDate
  3. Para PayU es importante que envíes el código postal junto con las direcciones de facturación y de envío
  4. Como comercio debes tener dirección física en México para poder usar nuestra API de promociones, procesamiento sin CVV2 y meses sin intereses
