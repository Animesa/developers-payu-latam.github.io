  • Es importante que en la petición XML/JSON, envies los datos del usuario que está realizando la compra. Con esta información, nuestro módulo antifraude tendrá mayores criterios para realizar la correcta validación de las transacciones.
  • La variable dniNumber hace referencia al documento público que contiene datos de identificación personal de los ciudadanos en cada país.
  • No solo existe la página de confirmación para recibir los estados finales de las transacciones. También puedes usar nuestra API Reportes para consultar los estados de las transacciones enviadas a PayU.
  • Para validar tu formulario de captura de datos de tarjeta de crédito debes tener en cuenta lo siguiente:
    • Las tarjetas VISA comienzan con el número 4 y tienen una longitud de 16 dígitos
    • Las tarjetas MASTERCARD comienzan con el número 5 y tienen una longitud de 16 dígitos
    • Las tarjetas AMEX comienzan con el número 3 y tienen una longitud de 15 dígitos
    • La tarjeta de crédito debe ser real, esto lo puede validar con el algoritmo de luhn
    • La fecha de expiración de las tarjetas debe ser mayor a la fecha actual
    • Validar el formato del correo electrónico
    • La opción de autocompletar no debe estar habilitada para dicho formulario
    • El campo de número de tarjeta de crédito y código de seguridad no debe permitir el ingreso de letras o caracteres especiales
  • El comprador es la persona que desea adquirir un producto o servicio, mientras que el pagador es la persona que realiza el abono de dinero. Ejemplo: Un joven desea adquirir un videojuego a través de internet (comprador), pero al momento de realizar el pago su padre ingresa los datos de su tarjeta de crédito (pagador).
 
Mejores practicas por país

Ten en cuenta las siguientes recomendaciones de acuerdo al país en el que llevarás a cabo el desarrollo de tu integracion PayU.

Argentina
  • Medios de integración disponibles: Web Checkout, API, SDK.
 
Brasil
  • Medios de integración disponibles: Web Checkout, API, SDK.
 
Colombia
  • Medios de integración disponibles: Web Checkout, API, SDK.
  • Para pagos PSE (Transferencia Bancaria) ten en cuenta la siguiente información:
    • CUS: Codigo Unico de Seguimiento. Es una variable que se envía en la página de respuesta; este campo indica la referencia de pago que emplea el banco para identificar la transacción.
    • pseBank: Este campo indica el nombre del banco a través del cual PSE procesará la transacción. Se envía en la página de respuesta.
    • pseCycle: Es una variable que se envía en la página de respuesta, es el Identificador generado por PSE.
    • pseReference1 / pseReference2 / pseReference3: Son variables que se envían en la página de respuesta, son los números de referencias para pagos con PSE.
 
México
  • Medios de integración disponibles: Web Checkout, API, SDK.
  • Para usar meses sin intereses debes hacer uso de nuestra API de promociones.
  • Debes enviar la variable BirthDate.
  • Para PayU es importante que envíes el código postal junto con las direcciones de facturación y de envío.
  • Como comercio debes tener dirección física en México para poder usar nuestra API de promociones, procesamiento sin CVV2 y meses sin intereses.
 
Panamá
  • Medios de integración disponibles: Web Checkout, API, SDK.
 
Perú
  • Medios de integración disponibles: Web Checkout, API, SDK.
